Today we are going to learn about lists!

You have probably seen horizontal navigation lists around the interwebs and you didn't realize it was just a list with some style. Take the following horizontal list. Don't worry, I won't press charges.

<ul class="horizontal-list">
  <li>
    <a href="https://rubymentor.io">Home</a> 
  </li>
  <li>
    <a href="https://blog.rubymentor.io">Blog</a> 
  </li>
  <li>
    <a href="https://guides.rubymentor.io">Guides</a> 
  </li>
</ul>

Here is the same list but actually displayed as HTML instead of having the HTML rendered. By the way, I put that in a <pre> tag to make it keep the spacing and formatting.

  • Blog
  • Guides

Noticed how the natural display of a list is to do a veritcal, bullet point based ist. Now let's change it to be inline which is the first step to having a horizontal list. In the CSS pane to the left add an class:

.horizontal-list {
  display: inline;
  list-style: none;
}

We still aren't flowing horizontally. And we still have bullets depending on the browser. The next step would be to scope this class to ul and li:

ul.horizontal-list li {
  display: inline;
  list-style: none;
}

Wow, that is already looking better! But we need to add some padding between these elements now so they aren't jammed up into each other like cars on the LA freeways.

We will now add one more designator to this list to put some distance between the li elements:

ul.horizontal-list li {
  display: inline;
  margin-right: 1em;
}

Wow, that did something! What if you wanted to put pipe character seperators in between the list items?

Well, that would require us to use some pseudo elements, namely the :after designator. So add the following CSS now:

ul.horizontal-list li:after {
  content: " |";
  margin-left: 1em;
}

We are almost there! But we really didn't want to have a pipe after the last li element. We can fix this with another pseudo designator, this time the last-child and another after like so:

ul.horizontal-list li:last-child:after {
  content: "";
}
